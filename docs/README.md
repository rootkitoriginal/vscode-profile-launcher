## Contributing to Documentation

We welcome documentation improvements! When contributing:

### Guidelines
1. **Placement**: Use appropriate directory structure
2. **Format**: Write in Markdown with clear headings
3. **Examples**: Include practical code samples
4. **Links**: Cross-reference related documents
5. **Maintenance**: Keep content up-to-date with code

### Types of Contributions
- Fix typos and errors
- Add missing information
- Improve clarity and examples
- Create new guides for uncovered topics
- Add diagrams and visualizations
- Translate documentation (future)

### Documentation Standards
- Use clear, concise language
- Include code examples where relevant
- Add screenshots for UI features
- Keep examples up-to-date
- Follow existing style and structure
